API REST – Documentación

Introducción

Este API expone el catálogo de teléfonos en JSON. Lectura es pública; creación/edición/eliminación requieren sesión de administrador.

Endpoints principales

  • GET /api/phones — listado con filtros, orden y paginación.
  • GET /api/phones/<id> — detalle por ID.
  • POST /api/phones — crear (admin).
  • PUT /api/phones/<id> — actualizar (admin).
  • DELETE /api/phones/<id> — eliminar (admin).
  • GET /api/compare?ids=1,2 — comparación JSON por IDs.

Listado con filtros

Parámetros: q (búsqueda), min_price, max_price, sort (id|name|price), order (asc|desc), page, page_size (1–100), fields (CSV de campos).

GET /api/phones?q=samsung&min_price=500&max_price=900&sort=price&order=asc&page=1&page_size=5 Response 200 { "items": [ {"id":1,"name":"Samsung ...","price":799.99, ...} ], "meta": {"page":1,"page_size":5,"total":10,"pages":2,"has_next":true,"has_prev":false}, "links": {"self":"http://localhost:5000/api/phones?q=samsung&..."} }

Seleccionar campos: /api/phones?fields=id,name,price

Comparación de modelos

Envíe al menos dos IDs separados por coma.

GET /api/compare?ids=1,2 Response 200 { "phones": [ {"id":1,"name":"Samsung ..."}, {"id":2,"name":"iPhone ..."} ], "differences": { "price": [799.99, 999.99], "storage": ["256GB", "128GB"] } }

Ejemplos rápidos

Fetch (frontend)

fetch('/api/phones?q=pro&sort=price&order=desc') .then(r => r.json()) .then(data => console.log(data.items))

cURL (crear como admin)

curl -X POST http://127.0.0.1:5000/api/phones \ -H "Content-Type: application/json" \ -d '{"name":"Demo","description":"Desc","ram":"8GB","storage":"128GB","camera":"50MP","battery":"4500mAh","price":599.99,"image_url":"https://..."}'

Errores y validaciones

  • 404: recurso no encontrado.
  • 403: operación requiere sesión de admin.
  • 400: parámetros inválidos (paginación, precio, IDs).
